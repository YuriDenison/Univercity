Ревью для проекта Priorityqueue.

Замечания:
1. Проект должен запускаться, ну или хотя бы не запускаться, но при этом не проходить юнит тесты.
2. Не нужно писать finally { //nothing here }
3. Тип, передаваемый генерику не может быть int, а должен быть Integer
4. Входные данные не должны меняться местами без необходимости.
5. Поля лучше задавать вначале класса.
6. Методы должны разделяться пустой строкой.
7. Комментарии типа "// constructor" только усложняют чтение кода, поэтому большинство "очевидных" комментариев можно не писать.
8. Незачем делать throws сразу на все исключения из Exception, ведь используется только одно (EmptyQueueException)
